# ESP32 Patrol Car

这是一个基于 ESP-IDF 的 ESP32 巡逻小车项目。当前 `main/` 目录已经包含超声波测距、四路巡线传感器与双路电机驱动的基础代码，可作为后续避障、循迹与速度闭环控制的开发基础。

## 当前代码结构

`main/` 目录包含以下文件：

- `main/main.cpp`：程序入口，初始化传感器并周期性输出测距与巡线数据。
- `main/HC_SR04.hpp`：HC-SR04 超声波传感器驱动。
- `main/LF04.hpp`：LF04 巡线传感器模板类，支持 1~8 路输入。
- `main/D153B.h`：D153B/TB6612 电机驱动接口定义。
- `main/D153B.cpp`：D153B/TB6612 电机驱动实现，包含 PWM、编码器与电压检测。
- `main/CMakeLists.txt`：ESP-IDF 组件注册配置。

## 已实现功能

### 1. 超声波测距

`HC_SR04.hpp` 中实现了 `UltrasonicSensor` 类：

- 使用 `TRIG` 输出 10us 触发脉冲。
- 使用 `ECHO` 引脚双边沿中断测量脉宽。
- 通过脉宽换算距离，最大值限制为 `400cm`。

当前在 `main/main.cpp` 中的引脚定义：

- `TRIG`: `GPIO_NUM_27`
- `ECHO`: `GPIO_NUM_35`

### 2. 四路巡线传感器

`LF04.hpp` 中实现了 `LineSensor<N>` 模板类：

- 支持 `1~8` 路数字输入。
- `read()` 可一次性读取所有通道并返回位掩码。
- `read(index)` 可读取单个探头状态。
- 状态定义：`Black = false`，`White = true`。

当前在 `main/main.cpp` 中使用了 4 路配置：

- `DH1`: `GPIO_NUM_32`
- `DH2`: `GPIO_NUM_33`
- `DH3`: `GPIO_NUM_4`
- `DH4`: `GPIO_NUM_39`

代码中按 `{DH4, DH3, DH2, DH1}` 顺序构造传感器数组，因此输出位序与物理接线顺序需要结合实际安装方向确认。

### 3. 电机驱动、编码器与电压检测

`D153B.h` 与 `D153B.cpp` 中实现了 `MotorDriver` 类，适用于基于 TB6612 方案的双路电机驱动板：

- 双路电机正反转与刹车控制。
- 基于 LEDC 的 PWM 调速。
- 基于 PCNT 的双编码器计数。
- 基于 ADC oneshot 的电源电压采样。

#### 主要接口

- `init()`：初始化 GPIO、PWM、编码器和 ADC。
- `set_speed(channel, speed)`：设置速度，范围 `-100 ~ 100`。
- `set_raw(channel, dir, duty)`：直接设置方向与占空比。
- `brake(channel)`：刹车。
- `standby(enable)`：控制驱动待机。
- `get_encoder_count(channel)`：读取编码器计数。
- `clear_encoder(channel)`：清零编码器计数。
- `read_power_voltage()`：读取电源电压，单位为 mV。

#### 驱动内部参数

- PWM 频率：`10kHz`
- PWM 分辨率：`10bit`
- 占空比上限：`1023`
- 编码器计数范围：`-20000 ~ 20000`
- 电压检测分压比：`1/11`

### 4. 当前主程序行为

`main/main.cpp` 当前逻辑较简单，主要用于硬件联调：

1. 输出 `Hello World!`
2. 初始化超声波传感器
3. 初始化四路巡线传感器
4. 每 500ms 串口输出：
   - 当前时间戳
   - 超声波测得距离
   - 巡线传感器整体位图值
   - 4 路单独状态

目前 `main.cpp` **尚未接入电机驱动逻辑**，`MotorDriver` 已实现但还没有在入口中实例化和调用。

## 构建方式

`main/CMakeLists.txt` 当前通过以下方式注册组件源文件：

- 自动收集 `main/` 下的 `*.cpp` 与 `*.hpp`
- 通过 `idf_component_register` 注册为 ESP-IDF 组件

如果你已经安装并配置好 ESP-IDF，可使用常规流程构建与烧录：

```bash
idf.py build
idf.py -p <PORT> flash monitor
```

## 后续开发建议

你提到**后续还会添加 PID 算法**，这与当前代码结构是匹配的，推荐按下面方向继续推进：

### 1. 先接入电机实跑

建议先在 `main.cpp` 中实例化 `MotorDriver`，完成：

- 左右电机正转/反转测试
- 编码器计数方向确认
- 电池电压读取校验

### 2. 增加编码器测速

在已有 `get_encoder_count()` 基础上，增加固定周期测速逻辑：

- 每隔固定时间读取编码器增量
- 换算为轮速或 RPM
- 作为 PID 输入

### 3. 引入 PID 速度控制

后续可增加 PID 控制器，实现：

- 目标速度设定
- 左右轮独立闭环
- 根据编码器反馈动态修正 PWM

推荐优先做：

- 单轮速度环 PID
- 双轮速度一致性控制
- 再叠加巡线偏差修正

### 4. 叠加巡线与避障逻辑

在速度环稳定后，再结合：

- 巡线传感器输出做方向修正
- 超声波距离做减速/停车/绕障策略

这样可以避免“感知、控制、执行”同时调试带来的干扰。

## 注意事项

- `HC_SR04.hpp` 中安装/卸载 GPIO ISR 服务时要注意是否会与其他 GPIO 中断模块冲突。
- `LineSensor` 读取到的黑白逻辑依赖传感器模块输出定义，实际使用前建议实测确认。
- `MotorDriver::standby(false)` 当前表示初始化后拉低待机脚，后续接线时应确认驱动板 `STBY` 的有效电平是否与预期一致。
- `main.cpp` 当前只做传感器打印验证，离完整小车控制程序还有一定距离。

## 总结

当前仓库已经具备巡逻小车的基础硬件驱动能力：

- 能测距
- 能读巡线状态
- 能控制双电机
- 能读取编码器和电源电压

下一阶段最值得优先完成的是：**电机接线联调 + 编码器测速 + PID 速度闭环**。这些能力补齐后，再继续实现稳定的巡线和避障控制会更顺利。
